# Digital Downloads Module

File delivery via secure, expiring download tokens. Associates downloadable files with products and generates single-use or limited-use tokens for order fulfillment. Supports batch token creation for orders with multiple digital products.

## Data models

- **downloadableFile**: id, productId, name, url (storage URL), fileSize?, mimeType?, isActive, createdAt, updatedAt
- **downloadToken**: id, token (UUID), fileId, orderId?, email, maxDownloads?, downloadCount, expiresAt?, revokedAt?, createdAt, updatedAt

## Token validation (useToken)

Checks in order:
1. Token not found → `{ ok: false, reason: "Token not found" }`
2. `revokedAt` is set → `{ ok: false, reason: "Token revoked" }`
3. `expiresAt < now` → `{ ok: false, reason: "Token expired" }`
4. `downloadCount >= maxDownloads` → `{ ok: false, reason: "Download limit reached" }`
5. Increment `downloadCount`, return `{ ok: true, file, token }`

## Batch token creation

`createTokenBatch({ fileIds, email, orderId?, maxDownloads?, expiresAt? })` creates one token per file ID. All tokens share the same email, orderId, maxDownloads, and expiresAt. Used for order processing with multiple digital products.

## Exports (for inter-module contracts)

Types: `DownloadableFile`, `DownloadToken`, `DigitalDownloadsController`

## Security

- `/downloads/me` requires authenticated session — email derived from `ctx.context.session.user.email`
- `/downloads/:token` is public (token is the secret) — no session required
- `/downloads/product/:productId` is public — returns only metadata (name, fileSize, mimeType), excludes download URL
- Admin endpoints protected by `createAdminEndpoint`

## Patterns

- Token value is a UUID generated by `crypto.randomUUID()` — distinct from the record `id`
- `getTokenByValue(token)` searches by the `token` field; `getToken(id)` searches by record `id`
- `revokeTokenById(id)` revokes by record ID; `revokeToken(value)` revokes by token value
- Controller key uses hyphen: `"digital-downloads"` — valid JS bracket notation
- `exactOptionalPropertyTypes` compatible: optional fields use conditional spreads
- The `url` field stores the raw storage URL — the application layer handles redirects or signed URLs
