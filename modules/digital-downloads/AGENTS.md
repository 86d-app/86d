# Digital Downloads Module

File delivery via secure, expiring download tokens. Associates downloadable files with products and generates single-use or limited-use tokens for order fulfillment.

## Structure

```
src/
  index.ts          Factory: digitalDownloads(options?) => Module
  schema.ts         Models: downloadableFile, downloadToken
  service.ts        DigitalDownloadsController interface + types
  service-impl.ts   DigitalDownloadsController implementation
  endpoints/
    store/          Customer-facing
      use-download.ts           GET  /downloads/:token (consume token, get file URL)
      list-my-downloads.ts      GET  /downloads/me?email= (list tokens by email)
    admin/          Protected (store admin only)
      list-files.ts             GET    /admin/downloads/files
      create-file.ts            POST   /admin/downloads/files/create
      delete-file.ts            DELETE /admin/downloads/files/:id/delete
      list-tokens.ts            GET    /admin/downloads/tokens
      create-token.ts           POST   /admin/downloads/tokens/create
  __tests__/
    service-impl.test.ts    35 tests
```

## Data models

- **downloadableFile**: id, productId, name, url (storage URL), fileSize?, mimeType?, isActive, createdAt, updatedAt
- **downloadToken**: id, token (UUID), fileId, orderId?, email, maxDownloads?, downloadCount, expiresAt?, revokedAt?, createdAt, updatedAt

## Token validation (useToken)

Checks in order:
1. Token not found → `{ ok: false, reason: "Token not found" }`
2. `revokedAt` is set → `{ ok: false, reason: "Token revoked" }`
3. `expiresAt < now` → `{ ok: false, reason: "Token expired" }`
4. `downloadCount >= maxDownloads` → `{ ok: false, reason: "Download limit reached" }`
5. Increment `downloadCount`, return `{ ok: true, file, token }`

## Exports (for inter-module contracts)

Types exported: `DownloadableFile`, `DownloadToken`, `DigitalDownloadsController`

## Security

- `/downloads/me` requires authenticated session — email derived from `ctx.context.session.user.email`
- `/downloads/:token` is public (token is the secret) — no session required

## Patterns

- Token value is a UUID generated by `crypto.randomUUID()` — distinct from the record `id`
- `getTokenByValue(token)` searches all records by the `token` field value (not `id`)
- Controller key uses hyphen: `"digital-downloads"` — valid JS bracket notation
- `exactOptionalPropertyTypes` compatible: optional fields use conditional spreads in createFile/createToken
- `findMany` uses spread pattern for optional take/skip
- The `url` field stores the raw storage URL — the application layer handles redirects or signed URLs
