# Registry Package

Git-based module registry for 86d. Resolves, fetches, and caches modules from local workspace, GitHub repositories, or npm packages.

## Structure

```
src/
  index.ts          Public API re-exports
  types.ts          Zod schemas + TypeScript types (manifest, specifier, config)
  specifier.ts      Parse module specifier strings into structured objects
  resolver.ts       Resolve config → concrete module list (local, registry, GitHub, npm)
  fetcher.ts        Download modules from GitHub tarballs or npm
  manifest.ts       Build registry.json from local modules/ directory
  config.ts         Read and normalize store config.json
  __tests__/        52 tests covering specifier, resolver, manifest, config, types, fetcher
```

## Module Specifier Formats

```
"products"                            → official (local or registry)
"@86d-app/products"                   → official (local or registry)
"github:owner/repo/modules/custom"    → GitHub subpath
"github:owner/repo/path#v2.0"         → GitHub with ref
"npm:@acme/module"                    → npm package
"npm:@acme/module@^1.0.0"             → npm with version
```

## Config Format

```json
{
  "modules": "*",
  "modules": ["@86d-app/products", "github:owner/repo/modules/custom"],
  "registry": "https://...registry.json",
  "moduleOptions": { "@86d-app/cart": { "maxItemsPerCart": 50 } }
}
```

- `"*"` or omitted → all local workspace modules
- Array → explicit list with missing module warnings

## Key Functions

- `parseSpecifier(raw)` → `ModuleSpecifier` (source, name, packageName, repo, path, ref)
- `resolveModules(config, { root, manifest? })` → `ResolvedModule[]`
- `fetchModule(spec, root, manifest?)` → `FetchResult` (downloads from GitHub/npm)
- `buildManifest(root)` → `RegistryManifest` (scans modules/ directory)
- `readStoreConfig(path)` → `StoreConfig`
- `getLocalModuleNames(root)` → `string[]`

## Registry Manifest (`registry.json`)

Generated by `scripts/generate-registry.ts`. Indexes all modules with metadata:
- name, description, version, category, path, requires
- hasStoreComponents, hasAdminComponents, hasStorePages

## Integration Points

- **`scripts/generate-modules.ts`** — uses `resolveModules()` + `fetchModule()` at buildtime to resolve all modules from config, fetch missing ones, then generate the store's import files
- **`packages/cli/src/commands/module.ts`** — uses `parseSpecifier()`, `fetchModule()`, `getLocalModuleNames()`, `readLocalManifest()` for `86d module add/list/search/info`
- **`scripts/generate-registry.ts`** — uses `buildManifest()` to generate `registry.json`

## Gotchas

- `exactOptionalPropertyTypes` is enabled — use conditional assignment for optional props
- GitHub fetching uses tarball API; set `GITHUB_TOKEN` for private repos or rate limits
- npm modules install to `node_modules/`, not `modules/`
- The resolver checks local workspace first before falling back to registry
